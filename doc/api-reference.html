    pocmonad v0.1.0 API Reference

    Modules

    Pocmonad.Happy

    This is the Happy module, Ok, so I was just trying to find a nice way (beautiful syntax, yet flexible enough) to handle errors in elixir. Handling :ok/:error like tuples without lots of if/cases

    Pocmonad.MonadEx

    This is the MonadEx module, MonadEx introduces monads into Elixir. Monads encapsulate state and control the flow of code. A monad’s bind operation is similar, but more powerful, than Elixir’s built-in pipelines

    Pocmonad.Ok

    This is the OK module, Elegant error/exception handling in Elixir, with result monads

    Pocmonad.Opus

    This is the Opus module, A framework for pluggable business logic components

    Pocmonad.Opus.Try1
    • input {:ok, any} call returns {:ok, any};
    Pocmonad.Opus.Try2
    • input {:ok, any} call returns {:error, Reason}
    Pocmonad.Rop

    This is the Rop module, The Elixir Pipeline with potatoes

    Pocmonad.RopMacro

    This is the RopMacro module, The Elixir Pipeline with potatoes and macros

    Pocmonad.With

    This is the With module, As we are now able to parse commands, we can finally start implementing the logic that runs the commands. Let’s add a stub definition for this function for now

    Pocmonad.Wormhole

    This is the Wormhole module, Wormhole captures anything that is emitted out of the callback (return value or error reason) and transfers it to the calling process in the form {:ok, state} or {:error, reason}
